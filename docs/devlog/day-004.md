# Day 004: Move Parsing

## Summary

Added a proper move representation and UCI-style long algebraic move parsing for local debugging and future move-generation work.

## Move Representation

`Move` stores:

- source square
- target square
- optional promotion piece

Promotion is limited to queen, rook, bishop, and knight. King and pawn promotions are rejected by the parser and validated promotion constructor.

## UCI-Style Move Parsing

Moves can now be parsed from strings such as:

```text
e2e4
g1f3
e7e8q
e7e8Q
```

Normal moves must be four characters. Promotion moves must be five characters. Uppercase promotion input is accepted, and serialization normalizes promotion pieces to lowercase.

The parser validates move-string structure only. It does not check whether the move is legal in a position or whether the source square contains a piece.

## Promotion Support

Allowed promotion characters:

```text
q r b n
```

Promotion targets are required to be on rank 1 or rank 8. This is still structural validation, not full chess legality.

## CLI Move Debug Command

Added:

```bash
cargo run -- move --parse e2e4
cargo run -- move --parse e7e8q
```

Example output:

```text
from: e7
to: e8
promotion: queen
uci: e7e8q
```

## Local Checks

`make check` now includes the move parser smoke commands:

```bash
cargo run -- move --parse e2e4
cargo run -- move --parse e7e8q
```
