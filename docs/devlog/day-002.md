# Day 002: FEN Parsing And Local Checks

## Summary

Implemented structural FEN parsing and added a repeatable local validation workflow for daily development.

## What Was Implemented

- `Position` stores:
  - piece placement on 64 squares
  - side to move
  - castling rights
  - en passant target square
  - halfmove clock
  - fullmove number
- `CastlingRights` represents `KQkq` availability.
- `FenParseError` reports malformed FEN structure.
- `STARTING_POSITION_FEN` defines the standard starting position.
- `Square::from_algebraic` supports square parsing for FEN fields and tests.

The parser validates FEN structure but does not validate chess legality.

## Local Checks

Added `scripts/local-check.sh` and `Makefile` targets for common checks:

```bash
make fmt
make clippy
make test
make build
make smoke
make check
```

Run the full validation workflow before every commit:

```bash
make check
```

## Intentionally Not Implemented Yet

- Legal move generation
- Search
- Evaluation
- UCI protocol handling
- Real perft execution
- Chess legality validation for parsed FEN positions

Add board mutation support for make/unmake move scaffolding, still without legal move generation.
