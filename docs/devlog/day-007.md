# Day 007: Pseudo-Legal Sliding Moves

## Summary

Added pseudo-legal sliding move generation for bishops, rooks, and queens, plus CLI debug commands for inspecting those moves from FEN positions.

## Sliding Move Generation Design

Sliding pieces use shared ray traversal. For each direction, the generator walks one square at a time until it:

- reaches the edge of the board
- hits a friendly piece and stops without adding that square
- hits an opponent piece, adds that capture, and stops
- sees an empty square, adds that move, and continues

This keeps bishop, rook, and queen behavior consistent and avoids unsafe code or unchecked board wrapping.

## Bishop Behavior

Bishops move diagonally in four directions. Tests cover center mobility, corner mobility, friendly blockers, opponent captures that stop the ray, edge handling, and the blocked starting position.

## Rook Behavior

Rooks move orthogonally in four directions. Tests cover center mobility, corner mobility, friendly blockers, opponent captures that stop the ray, edge handling, and the blocked starting position.

## Queen Behavior

Queens combine bishop-like diagonal rays and rook-like orthogonal rays. Tests cover center mobility, corner mobility, friendly blockers, opponent captures that stop rays, combined movement, and the blocked starting position.

## CLI Usage

```bash
cargo run -- movegen bishops --fen "8/8/8/3B4/8/8/8/8 w - - 0 1"
cargo run -- movegen rooks --fen "8/8/8/3R4/8/8/8/8 w - - 0 1"
cargo run -- movegen queens --fen "8/8/8/3Q4/8/8/8/8 w - - 0 1"
```

Each command prints normalized UCI moves followed by a total count.
