# Day 011 - Check Detection and Legal Move Filtering

## Summary

Added the first legal move layer on top of pseudo-legal move generation. The implementation stays intentionally simple: generate pseudo-legal moves, apply each candidate, and keep only moves that leave the moving side's king safe.

## Implemented

- Added `find_king(&Position, Color) -> Option<Square>`.
- Added `is_in_check(&Position, Color) -> bool`.
- Added `generate_legal_moves(&Position) -> Vec<Move>`.
- Legal filtering rejects moves that expose or leave the moving side's king in check.
- Legal filtering skips pseudo-legal moves that would capture the opponent king.
- Added CLI inspection through:

```bash
rookforge movegen legal --fen startpos
```

## Correctness Approach

The legal generator favors clarity over speed:

1. Generate pseudo-legal moves for the side to move.
2. Apply each candidate move to a cloned position.
3. Check whether the moving side is in check in the resulting position.
4. Keep only king-safe moves.

This is slower than a mature engine design but gives the project a clean correctness baseline before perft.

## Tests Added

- King lookup for both colors
- Missing king behavior
- Not-in-check position
- Rook, bishop, queen, knight, pawn, and adjacent-king check detection
- Filtering moves that expose the moving side's king
- Blocking a check
- Moving the king out of check
- Capturing a checking piece
- Avoiding opponent-king captures
- Starting position legal move count of 20
- Empty board legal move count of 0
- CLI unit and integration coverage for `movegen legal`

```

Run the complete validation workflow before committing:

```
