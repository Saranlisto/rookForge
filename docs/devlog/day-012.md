# Day 012 - Basic Perft

## Summary

Added basic recursive perft on top of legal move generation. Perft counts legal move-tree nodes and gives the project its first repeatable move-generation validation mechanism.

## Implemented

- Added `perft(&Position, depth: u32) -> u64`.
- Depth 0 returns 1 for the current node.
- Depth 1 returns the number of legal moves.
- Deeper depths recursively apply legal moves and sum child node counts.
- Added CLI execution through:

```bash
rookforge perft --fen startpos --depth 1
rookforge perft --fen startpos --depth 2
```

Example output:

```text
fen: startpos
depth: 2
nodes: 400
```

## Why Perft Matters

Perft is a correctness tool for move generation. It does not search for good chess moves. Instead, it verifies that the legal move tree has the expected number of nodes from known positions.

## Tests Added

- Start position depth 0 = 1
- Start position depth 1 = 20
- Start position depth 2 = 400
- Empty board depth 1 = 0
- King-only depth 1 count
- Pinned-piece depth 1 count that excludes illegal king exposures
- CLI unit and integration coverage for `rookforge perft --fen startpos --depth 2`

## Local Checks

The local smoke workflow now includes:

```bash
cargo run -- perft --fen startpos --depth 1
cargo run -- perft --fen startpos --depth 2
```

Run the complete validation workflow before committing:

```bash
make check
```
