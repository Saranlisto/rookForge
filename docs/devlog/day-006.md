# Day 006: Pseudo-Legal Knight And King Moves

## Summary

Added pseudo-legal move generation for knights and one-square king moves, along with CLI debug commands for inspecting them from FEN positions.

## Knight Move Generation

Knights now generate the standard eight L-shaped offsets when available. The generator:

- emits moves only for the side to move
- skips off-board targets
- skips friendly-occupied targets
- includes opponent-occupied targets as captures
- does not consider check, pins, or king safety

## King Move Generation

Kings now generate one-square horizontal, vertical, and diagonal moves. The generator:

- emits moves only for the side to move
- skips off-board targets
- skips friendly-occupied targets
- includes opponent-occupied targets as captures
- does not check whether the destination is attacked
- does not generate castling

## Shared Occupancy Handling

Knight and king generation share leaper helper logic for target squares: empty squares are allowed, opponent pieces are capturable, and friendly pieces block the move.

## CLI Usage

```bash
cargo run -- movegen knights --fen startpos
cargo run -- movegen kings --fen "8/8/8/8/4K3/8/8/8 w - - 0 1"
```

Each command prints normalized UCI moves followed by a total count.

## Local Checks

`make check` now includes:

