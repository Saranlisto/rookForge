# Day 010 - Attack Detection

## Summary

Added the first attack-detection layer for the engine core. The new API answers whether a target square is attacked by a selected color without changing the position and without depending on side to move.

## Implemented

- Added `is_square_attacked(&Position, Square, Color) -> bool`.
- Added pawn attack detection using capture geometry, not pawn push movement.
- Added knight and king attack detection using fixed deltas.
- Added bishop and rook ray attacks with blocker handling.
- Added queen attack coverage through diagonal and orthogonal slider checks.
- Added CLI inspection through:

```bash
rookforge attacks --fen "<FEN|startpos>" --square e4 --by black
rookforge attacks --fen "<FEN|startpos>" --square e4 --by white
```

The command prints:

```text
square: e4
by: black
attacked: true
```

## Tests Added

- White pawn attacks
- Black pawn attacks
- Pawn edge-file behavior
- Knight attacks from center and edge squares
- King adjacency attacks
- Bishop, rook, and queen attacks
- Slider blocker behavior
- Empty-board non-attacks
- Side-to-move independence
- CLI unit and integration coverage for `rookforge attacks`

## Local Checks

The local smoke workflow now includes attack-inspection commands:

```bash
cargo run -- attacks --fen "4r3/8/8/8/4K3/8/8/8 w - - 0 1" --square e4 --by black
cargo run -- attacks --fen startpos --square e4 --by black
cargo run -- attacks --fen startpos --square e4 --by white
```

Run the complete validation workflow before committing:

```bash
make check
```
