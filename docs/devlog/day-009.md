# Day 009: Basic Move Application

## Summary

Added basic position mutation through a new `apply_move` API. The function returns a new `Position` instead of mutating the input position, keeping early move-application behavior easy to test and reason about.

## Move Application Behavior

`apply_move` now:

- reads the moving piece from the source square
- returns an error if the source square is empty
- clears the source square
- writes the moving or promoted piece to the target square
- replaces any captured target piece
- flips side to move
- updates the halfmove clock
- updates the fullmove number
- clears or sets the en passant target square
- updates castling rights affected by king or rook moves and original-square rook captures

This is structural move application only. It does not check whether the move is legal or pseudo-legal.

## Supported Move Types

Supported:

- quiet moves
- captures
- promotions to queen, rook, bishop, or knight

Promotion keeps the moving pawn's color. Promotion requested for a non-pawn returns an error.

## Move Counters

Halfmove clock:

- resets to `0` for pawn moves
- resets to `0` for captures
- increments for quiet non-pawn moves

Fullmove number:

- stays unchanged after a White move
- increments after a Black move

## Castling Rights

Castling moves are not implemented yet, but castling rights are updated when:

- a king moves
- a rook moves from `a1`, `h1`, `a8`, or `h8`
- a rook is captured on `a1`, `h1`, `a8`, or `h8`

## En Passant Field

En passant capture is not implemented yet. The en passant target is cleared after most moves and set after simple two-square pawn moves such as `e2e4` or `d7d5`.

## CLI Usage

```bash
cargo run -- apply --fen startpos --move e2e4
cargo run -- apply --fen "8/8/8/8/8/8/4P3/8 w - - 0 1" --move e2e4
```

The command prints the resulting FEN followed by a human-readable board.

## Local Checks

`make check` now includes:

```bash
cargo run -- apply --fen startpos --move e2e4
```

## Intentionally Not Implemented Yet

- Legal move filtering
- Check detection
- Castling execution
- En passant capture
- Unapply/reversible move history
- Real perft
- Search
- Evaluation
- UCI engine protocol

## Next Step

Add reversible move history and unapply scaffolding so perft and search can safely traverse move trees.
